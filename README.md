# The Digital Income Blueprint — site rebuild

Recreated landing page + real Yoco inline checkout (no more WhatsApp redirect).

## What's inside
- `index.html` — the full landing page (self-contained, no build step)
- `api/charge.js` — Vercel serverless function that charges the card via Yoco's Charges API, then issues a signed, time-limited download token
- `api/download.js` — Vercel serverless function that only serves the PDF if it receives a valid, unexpired token from a real successful charge
- `private/Digital_Income_Blueprint.pdf` — the actual guide, kept out of any public path so it can only be reached through `api/download.js`
- `vercel.json` — function config, including bundling the private PDF with the download function

**The PDF is gated:** it is not a public static file anymore. The only way to fetch it is `/api/download` with a valid `chargeId` + `expires` + `token`, which are generated by `api/charge.js` only after Yoco confirms the card was actually charged. Each link expires 1 hour after purchase.

## 1. Add your Yoco keys

**Public key** (safe to expose in the browser):
Open `index.html`, find this line near the bottom, and replace it with your real public key:
```js
const YOCO_PUBLIC_KEY = "pk_test_xxxxxxxxxxxx"; // <-- your real public key
```
Use your `pk_test_...` key while testing, then swap to `pk_live_...` when you go live.

**Secret key** (must never be in frontend code — used only by `api/charge.js`):
1. Deploy this project to Vercel (or open your existing Vercel project settings).
2. Go to **Project Settings → Environment Variables**.
3. Add a variable:
   - Name: `YOCO_SECRET_KEY`
   - Value: your `sk_test_...` (or `sk_live_...` when live) secret key from the Yoco dashboard.
4. Redeploy — the function reads it automatically, it's never sent to the browser.

## 2. Add the real PDF
Replace the file inside the `private/` folder with your actual guide, keeping the same filename:
```
private/Digital_Income_Blueprint.pdf
```
Do not move it to the project root or into a `public/` folder — it must stay in `private/` so it's never served as a plain static file. It's only ever returned through `/api/download`, and only with a valid token from a real payment.

## 3. Deploy
Easiest path — Vercel CLI:
```bash
npm i -g vercel
cd dib-site
vercel
```
Or drag-and-drop the folder into a new Vercel project in the dashboard, then add the environment variable from step 1 before your first real deploy.

## 4. Test a payment
Yoco test cards: https://developer.yoco.com/online/testing — use a test card with your `pk_test_...` / `sk_test_...` pair before switching to live keys.

## Notes
- The checkout modal collects name + email, opens Yoco's secure card popup, tokenizes the card client-side, then sends that token to `/api/charge` where the real charge happens server-side with your secret key. No card details ever touch your own server.
- Currently the guide is downloaded directly in-browser after payment. If you'd like it emailed automatically too, that needs an email service (e.g. Resend or SendGrid) wired into `api/charge.js` — happy to add that if you want it.
